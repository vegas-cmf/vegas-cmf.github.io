Application structure

Modules

Application based on Vegas CMF is separated on modules, that define controllers, models, forms and so on.. Each module has own config file and routes definition.
To create new module, make a directory in the app/module with the first upper character, for example: Foo. To register module in application create a Module.php file inside module directory.
The default form of Module file: 


namespace Foo;

class Module extends \Vegas\Mvc\ModuleAbstract
{
    public function __construct() {
        $this->namespace = __NAMESPACE__;
        $this->dir = __DIR__;
    }
}

All modules in application are loaded automatically and defined in app/config/modules.php file. In the development environment, list of modules is rebuilding for each request, so you do not have to care about it, when you create a new module. In the production environment app/config/modules.php file is created only when does not exist, so when you update your production instance, keep in mind to remove that file, to force renewal.
In addition each module might have own configuration file, located in module-name/config/config.php file.

Module structure

Each module beside Module.php file and own configuration, can contain controllers, models, views, services and components. All of these parts are being autoloaded in runtime, so you can reach them from every location inside the application. Check the following examples to see how to create them in proper way.

In case of controllers, Vegas CMF provides possibility to separate controllers, to keep DRY rule. Consider the following scenario:

Application should contain three parts: the frontend (end-user interface), the backend (administrators panel) and the dashboard (for customers). All of these areas will use the same data model and services. Each part will have another layout and views.

Thanks to Vegas CMF, the preceding scenario is really easy to implement.
In the controllers directory, make a three directories
  • frontend
  • backend
  • dashboard


The views directory should also have three directories
  • frontend
  • backend
  • dashboard

The whole module structure should look like as follows 


Foo/
    - config
    - controllers/
        - backend
        - dashboard
        - frontend
    - models
    - services
    - views/
        - backend
        - dashboard
        - frontend

Once we have a ready structure, we can start creating controllers and other stuff.
Lets make a HomeController for each sub module 


//controllers/backend

namespace Foo\Controllers\Backend;

use Vegas\Mvc\Controller\ControllerAbstract;

/**
 * Class HomeController
 * @package Foo\Controllers\Backend
*/
class HomeController extends ControllerAbstract
{
    //....
    public function indexAction()
    {
        $this->view->welcome = 'Welcome in backend';
    }
}



//controllers/dashboard

namespace Foo\Controllers\Dashboard;

use Vegas\Mvc\Controller\ControllerAbstract;

/**
 * Class HomeController
 * @package Foo\Controllers\Dashboard
*/
class HomeController extends ControllerAbstract
{
    //....
    public function indexAction()
    {
        $this->view->welcome = 'Welcome in dashboard';
    }

}



//controllers/frontend

namespace Foo\Controllers\Frontend;

use Vegas\Mvc\Controller\ControllerAbstract;

/**
 * Class HomeController
 * @package Foo\Controllers\Frontend
*/
class HomeController extends ControllerAbstract
{
    //....
    public function indexAction()
    {
        $this->view->welcome = 'Welcome in frontend';
    }
}

To make controllers accessible, lets define routes rules in the foo/config/routes.php file 


/** FRONTEND */
'home' => array(
    'route' =>  '/',
    'paths' => array(
        'module' => 'Foo',
        'controller' => 'Frontend\Home',
        'action' => 'index',

        'auth' => false
    ),
    'params' => array()
),
/** BACKEND */
'backend_home' => array(
    'route' =>  '/backend',
    'paths' => array(
        'module' => 'Foo',
        'controller' => 'Backend\Home',
        'action' => 'index',

        'auth' => false
    ),
    'params' => array()
),
/** DASHBOARD */
'dashboard_home' => array(
    'route' =>  '/dashboard',
    'paths' => array(
        'module' => 'Foo',
        'controller' => 'Dashboard\Home',
        'action' => 'index',

        'auth' => false
    ),
    'params' => array()
),


Now we can create some models and services which will be common for all controllers.
