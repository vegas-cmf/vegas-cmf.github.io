Command Line Interface

Create task

Command Line Interface allows to simply create scripts, which can be executed from system shell or via cron.
We can create a Task in three locations:

  • global application task - located in the "app/tasks/" directory
  • module task - located in the application module directory, eg. "app/modules/ModuleName/tasks"
  • library task - located in the vegas library installed via composer, eg. "vendor/vegas-cmf/core/Task

Task class must extend "Vegas\Cli\Task" class and implement method setOptions that defines all available task actions.
Name of task class must contain word "Task", eg. "CustomTask". 

class CustomTask extends \Vegas\Cli\Task
{
    /**
     * Available options
     *
     * @return mixed
     */
    public function setOptions()
    {
        $action = new \Vegas\Cli\Task\Action('test', 'Test action');

        //foo option
        $foo = new \Vegas\Cli\Task\Option('foo', 'f', 'Foo option. Usage app:custom test -f numberOfSth');
        $foo->setValidator(function($value) {
            if (!is_numeric($value)) return false;
            return true;
        });
        $foo->setRequired(true);
        $action->addOption($foo);
        $this->addTaskAction($action);
    }

    /**
     * Test action that shows number passed to "f" option and in the first argument
     */
    public function testAction()
    {
        $this->putText($this->getArg(0));
        $this->putText($this->getOption('f'));
    }
}

To run the preceding task enter this command in shell 

php cli/cli.php app:custom test 6789 -f 1234

It will print
6789
1234

app:custom means that task is a global application task, and it's located in the "app/tasks/CustomTask.php" file.
test it's a name of task action to execute.
6789 it's the first execution argument.
-f 1234 it's the value of option "f" ("foo").

To create a module-specific task, create the task file inside "tasks" directory, eg. in the "Example" module (app/modules/Example/tasks) 


namespace Example\Tasks;

class CustomTask extends \Vegas\Cli\Task
{
    /**
     * Available options
     *
     * @return mixed
     */
    public function setOptions()
    {
        $action = new \Vegas\Cli\Task\Action('test', 'Test action');

        //foo option
        $foo = new \Vegas\Cli\Task\Option('foo', 'f', 'Foo option. Usage app:custom test -f numberOfSth');
        $foo->setValidator(function($value) {
            if (!is_numeric($value)) return false;
            return true;
        });
        $foo->setRequired(true);
        $action->addOption($foo);
        $this->addTaskAction($action);
    }

    /**
     * Test action that shows number passed to "f" option and in the first argument
     */
    public function testAction()
    {
        $this->putText($this->getArg(0));
        $this->putText($this->getOption('f'));
    }
}

The only one different is that module-specific task has a namespace and can be execute as follows: 

php cli/cli.php app:example:custom test 6789 -f 1234

The output will be exactly the same like in the previous example.

The third type of task is a vegas-library task, that is located inside library namespace. For example in the core library, there is a task named \Vegas\Task\CacheTask which has an action clean and can be executed as follows: 

php cli/cli.php vegas:cache clean

vegas:cache means that task is located inside namespace \Vegas\Task. When task class is located in more deeper namespace, eg. \Vegas\Tool\Faker\Task\GeneratorTask, then name of command should look like: vegas:tool_faker:generator, so the second part it's a namespace seperated by underscore "tool_faker".
